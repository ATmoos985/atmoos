这是一份给自己看的 Markdown 备忘。

## 标题

标题用于组织结构，不要只为了放大文字而使用标题。

```markdown
# 一级标题

## 二级标题

### 三级标题
```

## 列表

普通清单用无序列表，流程和顺序用有序列表。

```markdown
- 一个要点
- 另一个要点

1. 第一步
2. 第二步
```

## 链接

```markdown
[链接文字](https://example.com)
```

内部内容后续可以补相对路径或笔记链接规则。

## 代码块

代码块尽量写语言类型，方便高亮。

````markdown
```ts
const title = 'Atmoos'
```
````

## 写作建议

先把内容写清楚，再考虑格式。

格式是为了降低阅读成本，不是为了让页面看起来复杂。
