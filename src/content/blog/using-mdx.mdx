MDX 可以在 Markdown 里使用组件，适合写更复杂的笔记。

## 适合使用 MDX 的情况

- 需要折叠补充信息。
- 需要标签页对比不同方案。
- 需要插入自定义卡片。
- 需要展示交互示例。

## 不适合使用 MDX 的情况

如果只是普通文章，Markdown 已经够用。

MDX 会增加维护成本，所以应该只在内容确实需要组件表达时使用。

## 当前规则

默认用 Markdown。

当一篇内容需要组件时，再改用 MDX。
