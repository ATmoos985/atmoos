## 发布流程

当前站点部署在 Vercel。正常发布流程是：

1. 本地完成内容或页面修改。
2. 执行类型检查和构建。
3. 提交代码并推送到 GitHub。
4. 由 Vercel 自动构建和发布。

## 发布前检查

每次发布前至少确认三件事：

- 首页能正常打开，头像和导航没有断。
- 杂想列表和笔记库目录能正常跳转。
- 构建没有类型错误和内容 schema 错误。

如果只是新增草稿，可以先设置 `draft: true`，确认不会出现在公开列表里。

## 后续改进

后续可以补一个发布清单，把搜索索引、站点地图、RSS、Open Graph 图片也纳入检查。
