## 使用原则

内容组件不是为了装饰页面，而是帮助读者更快理解信息。

推荐用法：

- 提示块：放注意事项、风险、结论。
- 折叠块：放补充说明、长代码、次要材料。
- 标签页：对比不同命令、不同平台或不同方案。
- 卡片列表：整理一组相关链接或工具。

## 写笔记时的判断

如果一段内容用普通 Markdown 就能讲清楚，不需要额外组件。

如果读者需要在几类信息之间切换，或者某段内容会干扰主线，再考虑组件。
