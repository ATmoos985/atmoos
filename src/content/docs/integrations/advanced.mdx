## 适用场景

交互组件适合放在需要探索的内容里，例如：

- 多步骤流程。
- 可展开的资料清单。
- 示例代码和结果对照。
- 项目进度或版本记录。

普通说明文不需要复杂交互，否则会分散注意力。

## 设计约束

交互内容要遵守三个原则：

1. 默认状态能看懂。
2. 展开后能提供额外价值。
3. 移动端不影响阅读。

## 暂定组件池

目前优先使用主题已有组件，不额外引入新的 UI 框架。

等内容规模增加后，再判断是否需要为知识库定制树形目录、双链引用或笔记关系图。
